This repository contains all the input files and data for the paper "Fast and accurate committor estimation for kinetics simulations"

The following folders contain the code, simulation setup, and models for the three systems (mueller potential, alanine dipeptide, and chignolin) presented in the manuscript.

```
.
├── mueller/
│   ├── overdamped
│   └── underdamped
├── alad/
├── chignolin/             
│   ├── 405_CV
│   └── 4_CV
└── README.md
```

Mueller (Python only)
The mueller potential calculations are completely self‑contained in Python. 
Running ./mueller/overdamped/sub.sh or ./mueller/underdamped/sub.sh executes the entire pipeline automatically.

Alad & Chignolin (NAMD)
NAMD 2.14 – compiled with TCL support.
All necessary CHARMM force field parameters and topology files are provided in ./alad/data0/ or ./chignolin/toppar/


Overall computational workflow

APC-based committor estimation (APC/): First, generate configurations ($\circ$ points) distributed across the configuration space. 
Subsequently, unbiased trajectory simulations are initiated from each sampled $\circ$ point to produce $n$ $\square$ points. 
Finally, committor values are estimated for all $\square$ points based on a discrete-time Markov chain constructed from analogue prediction. 
Running ./APC/sub.slurm executes the above three steps automatically. 

Neural network training (nn/): At the end of the APC algorithm, a neural network is trained based on these computed committor values on $\square$ poitns to obtain a smooth representation of the committor. 

Milestoning analysis (milestoning/): Performing Milestoning simulations and analysis. 


Detailed procedure

APC
Step1: cd ./APC/
Step2: manually change the relevant paths in ./APC/sample_data/sample_cell.namd, ./APC/run_traj_data/run_traj.namd files, and sub.slurm ('current_dir=CoM/alad/APC', 'current_dir=CoM/chignolin/4_CV/APC', or 'current_dir=CoM/chignolin/405_CV/APC') to proper ones.
Step3: sbatch ./sub.slurm   ### note that the 'i=0' parameter in sub.slurm only runs for the 0-th iteration
                            ### demo output files are provided: ../demo-output/APC/run_traj_all.out and ../demo-output/APC/C_values.dat  
Step4: run the next iteration, please manually change 'i=0' to 'i=1' in ./sub.slurm
Step5: sbatch ./sub.slurm   ### demo output files are provided: ../demo-output/APC/run_traj_all.out, ../demo-output/APC/C_values.dat and ../demo-output/APC/aver_diff_c_values.txt
Step6: manually inspect the file ./APC/run_traj_$i/ana_data/aver_diff_c_values.txt to determine whether the predicted committor values have converged (i.e., all elements in the second column of aver_diff_c_values.txt are below 0.1)
Step7: repeat Steps 4-6 (remember the manually change 'i=$i' in Step 4) until the predicted committor values converge

nn
Step1: cd ./nn
Step2: cp ../APC/run_traj_$i/ana_data/C_values.dat ./
Step3: python3 ./nn.py  ### demo output files are provided: ../demo-output/nn/best_model.pth and ../demo-output/nn/network_params.npz 

milestoning
Step1: cd ./milestoning
Step2: generate initial configurations for the reactant and product milestones using conventional restrained sampling and adopt a simpler initialization strategy by directly collecting $\square$ points on a milestone for all other intermediate milestones (The configurations used in the paper are also provided)
Step3: cp ../nn/best_model.pth ./
Step4: python3 ./milestoning/data0/pth_to_tcl.py ### demo output files are provided: ../demo-output/milestoning/network_params.tcl
Step5: manually change the relevant paths in the shooting.namd file to proper ones
Step6: cp ./milestoning/data0/network_params.tcl ./milestoning/mile_$i
Step7: cd ./milestoning/mile_$i
Step8: sbatch ./milestoning/mile_$i/sub.slurm
Step9: python3 ./milestoning/data0/new_result.py   ### demo output file is provided: ../demo-output/milestoning/cm.out
                                                   ### extract the local transition information among milestones from shooting.log file generated by Step 8.
Step10: python3 ./milestoning/data0/MFPT.py   ### demo output file is provided: ../demo-output/milestoning/mfpt.txt
